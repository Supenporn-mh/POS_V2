# คำสั่ง Design System — POS D4-504 (Wise Canteen / UPOS)

ใช้คู่กับ `POS_D4504_TAP_TO_COLLECT_SPEC.md` — ไฟล์นี้คือ token กลาง ห้ามสร้าง hex/ค่าใหม่นอกตารางนี้ ถ้าต้องการสีเพิ่มให้กลับมาตกลงกันก่อน ไม่ใช่เดาเอาเอง

> อ้างอิงจาก token ชุดเดียวกับที่ extract จาก `style.css` ของโปรเจกต์ POS_V2 (โค้ดต้นฉบับ D4-504) ที่เคยใช้ทำ handheld version มาก่อน — ถ้าโปรเจกต์จริงมีการแก้ hex ไปแล้วหลังจากนั้น ให้แจ้งกลับมาเพื่ออัปเดตไฟล์นี้ให้ตรง

### 2.1 Mapping สถานะการจอง (5 สถานะ) → token

| Status | Label ไทย | Token สี |
|---|---|---|
| Confirmed | จองแล้ว | `--green` |
| Ready to collect | รอวันรับ | `--blue` |
| Collected | รับแล้ว | `--gray` |
| Missed | ไม่มารับ | `--red` |
| Cancelled | ยกเลิกแล้ว | `--red` |

### 2.2 Mapping edge case → token (สรุปจากทุกเคสที่ออกแบบไว้)

| ระดับ | เคส | Token |
|---|---|---|
| Info (เทา/น้ำเงิน) | ไม่พบการจอง, บัตรไม่รู้จัก, มาก่อนเวลา, ระบบออฟไลน์, แตะบัตรครั้งแรก(retry) | `--gray` หรือ `--blue-bg` |
| Warning (เหลือง) | แตะซ้ำ(รับไปแล้ว), จองคนละมื้อ, ยังไม่ชำระเงิน | `--amber` |
| Danger (แดง) | พ้นเวลารับ, บัตรถูกระงับ, ยกเลิกแล้ว, เครื่องอ่านขัดข้อง, บัตรชำรุด | `--red` |
| Success (เขียว) | รับอาหารสำเร็จ | `--green` |

หลักการ: **แดงสงวนไว้เฉพาะสิ่งที่บล็อก action หรือผิดปกติจริง** ส่วนเคสที่แค่ไม่ตรงเงื่อนไข (เช่น มาก่อนเวลา, จองคนละมื้อ) ใช้โทนเตือน/กลาง ไม่ใช่แดง เพื่อไม่ให้นักเรียนรู้สึกว่าทำผิด

### 2.3 Token เพิ่มเติม (เพิ่มหลัง audit ทั้งระบบ — ยืนยันกับทีมแล้ว)

ระบบ POS_V2 มีสีที่ใช้งานจริงมานานแล้วแต่ตารางเดิมไม่ครอบคลุม — เพิ่มเป็น token ทางการ ห้ามใช้ hex อื่นแทนสองสีนี้:

| Token | Hex | Role | ใช้กับ |
|---|---|---|---|
| `--purple` | `#AF52DE` | coupon / สถานะ "served" | badge คูปอง (แทน `badge-redeem` เดิม), สถานะ "served" ของ Kitchen/Food-serving, การ์ด feature "fi-purple" |
| `--purple-bg` | `#F5EEFF` | tint ของ purple | พื้นหลัง badge/การ์ดข้างต้น |
| `--emerald` | `#16A34A` | ยอดเงินออม/savings total | ตัวเลขส่วนลดที่ประหยัดได้ (`promo-applied-saving`), badge "พิมพ์แล้ว" ใน Kitchen |
| `--emerald-bg` | `#F0FDF4` | tint ของ emerald | พื้นหลัง badge/ปุ่มข้างต้น (border คู่กัน `#BBF7D0`) |

> หมายเหตุ: token คู่นี้ตรงกับที่ `CLAUDE.md` ของโปรเจกต์กำหนดไว้แล้ว (purple=coupon, emerald=savings) — เอกสารนี้แค่เพิ่มให้ตรงกัน ไม่ใช่ค่าใหม่ที่คิดขึ้นเอง

**ข้อยกเว้น — สี brand บุคคลที่สาม:** โลโก้/สีช่องทางชำระเงิน (Alipay, WeChat Pay, TrueMoney, พร้อมเพย์ ฯลฯ ในหน้า Money Management) **ไม่ต้องบังคับให้เข้า token ในไฟล์นี้** ใช้สี brand ทางการของผู้ให้บริการนั้นได้ตรงๆ

## 3. Typography

**Font-family (ยืนยันแล้ว):** `'IBM Plex Sans Thai', sans-serif` ใช้ทั้งระบบ — ไม่มีข้อยกเว้นแยกภาษา/หน้าจอ ที่ไหนยังหลุดเป็น `'Inter'`/`'Kanit'` (เช่นหน้า login) ให้แก้เป็นตัวนี้เมื่อถึง phase ที่แตะไฟล์นั้น

2 weight เท่านั้น: **400** (ข้อความทั่วไป, caption) และ **500** (หัวข้อ, label, ราคา, ชื่อ) — **ห้ามใช้ 300/600/700** ที่เจอในโค้ดเดิม (audit พบ 300×25, 600×30, 700×8 จุด) ให้ปัดเข้า 400 หรือ 500 ตามบริบทเมื่อแก้

| Scale | ขนาด | Weight | ใช้กับ |
|---|---|---|---|
| Display-XL (ยืนยันแล้ว) | 44px | 500 | ตัวเลขยอดเงินคงเหลือตัวใหญ่ (หน้าเช็คยอดเงินรายวัน/การจัดการเงิน) เท่านั้น |
| Display | 28px | 500 | ตัวเลขสรุปยอดใหญ่ (หน้าภาพรวม, การ์ดสรุป) |
| Title | 17px | 500 | ชื่อหน้าจอ, ข้อความสถานะหลัก ("รับอาหารสำเร็จ") |
| Subtitle | 15px | 500 | ชื่อนักเรียน |
| Body | 14px | 400 | รายการอาหาร, ข้อความอธิบาย |
| Label | 13px | 500 | badge, ปุ่ม, tab |
| Caption | 11px | 400 | เวลา, timestamp, sub-text |

> ตัวเลขในโค้ดเดิมที่เคยใหญ่กว่า Display-XL (audit เจอสูงสุดถึง 72px) ให้ปรับลงมาเป็น 44px ตอนแก้ไฟล์นั้นจริง — ห้ามมีขนาดที่ 3 สำหรับตัวเลขยอดเงิน

## 4. Spacing & Radius

| Token | ค่า | ใช้กับ |
|---|---|---|
| Radius — การ์ด/ปุ่มใหญ่ | 12–16px | การ์ดรายการอาหาร, ปุ่ม "ยืนยันรับอาหาร" |
| Radius — control เล็ก (ยืนยันแล้ว) | 10px | input, chip, ปุ่มเล็ก, date picker — มาตรฐานใหม่จากค่าที่ใช้บ่อยที่สุดในโค้ดเดิม (60 จุด) |
| Radius — badge/pill | 20px (full pill) | badge สถานะทุกตัว |
| Radius — modal/sheet | 20px (มุมกลม 4 ด้าน) | modal รายละเอียด, modal ยกเลิก |
| Gap ระหว่าง element | 8–12px | ระยะห่าง row ในรายการอาหาร, ระหว่างการ์ดสรุป |
| Padding การ์ด | 12–16px | การ์ดสรุปยอด, การ์ดรายการอาหาร |
| Padding หน้าจอ (ซ้าย-ขวา) | 16–24px | จอ 1 (มีพื้นที่กว้าง ไม่ใช่ handheld) |

## 5. Component states ที่ต้องมีทุกจุด

- ปุ่ม primary ต้องมี state: default / pressed / disabled (สีจาง 40% opacity)
- Badge ทุกตัวใช้ font-weight 500, ขนาด label scale, พื้นหลัง tint สีเดียวกับ role
- Input (รหัส supervisor, ช่องค้นหา) ใช้ `--border` เป็นขอบปกติ, เปลี่ยนเป็น `--red` เมื่อ error
- ทุกหน้าที่ auto-dismiss (idle timeout) ต้องมี transition fade ไม่ใช่ตัดหน้าทันที (ป้องกันความรู้สึกกระตุก)
